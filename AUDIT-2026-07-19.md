## Executive result

The mobile “crash and reload” was a forced service-worker takeover, not a canvas-memory failure. Returning to a backgrounded iOS tab triggered an update check; the new worker skipped its waiting phase, claimed the open tab, emitted `controllerchange`, and the registration code hard-reloaded the page. The lifecycle is now passive: an update waits until existing tabs close and takes effect on a normal future navigation (`app/public/register-sw.js:1-9`, `app/public/sw.js:12-22`). A regression ratchet prevents the forced lifecycle from returning (`app/scripts/audit-mobile-lifecycle.mjs:1-32`).

The route estate is otherwise healthy. All 182 generated routes were rendered at desktop, tablet, and mobile sizes: 125 public sitemap routes against production plus 57 deliberately noindex routes against the final local build. Across 546 viewport renders the audit found zero Axe violations, zero captured console errors, zero horizontal-overflow routes, and zero dark dead-zone risks.

## Confirmed defects repaired

### 1. Mobile foreground reload loop — resolved

The original registration performed four hazardous actions together: checking for updates every time the document became visible, sending `SKIP_WAITING`, claiming active clients, and reloading on `controllerchange`. WebKit reproduction proved that dispatching `controllerchange` increased the page-load count and changed the navigation type to `reload`.

The registration now performs only best-effort progressive registration (`app/public/register-sw.js:1-9`). The worker no longer calls `skipWaiting()`, `clients.claim()`, or handles a forced-activation message (`app/public/sw.js:12-22`). PWA caching, offline navigation fallback, and stale-while-revalidate assets remain intact (`app/public/sw.js:24-73`).

Final WebKit regression:

- First install: one registration, no forced controller takeover.
- Normal reload: worker controls the page.
- Synthetic `controllerchange`: page load count remains one.
- Full 12,049-pixel home-page scroll: six visible canvases, no horizontal overflow, no route error, and no console errors.

### 2. WebKit chunk-error handling could loop — resolved

Safari reports failed module imports as `Importing a module script failed.` The boundary now recognizes that exact error (`app/src/components/ErrorBoundary.tsx:21-32`). An automatic recovery reload is allowed only when its `sessionStorage` cooldown guard can be both read and written; blocked storage now displays the branded fallback instead of risking an unbounded refresh loop (`app/src/components/ErrorBoundary.tsx:42-68`).

Regression results:

- Storage available: exactly one recovery reload, then fallback if the chunk remains missing.
- Storage blocked: zero automatic reloads and an accessible branded fallback.

### 3. Ineffective analytics split — resolved

Event call sites statically imported the analytics module while the entry also dynamically imported it, so Vite correctly warned that the dynamic import could not split the module. Call sites now use a small lazy bridge that treats analytics as non-critical (`app/src/lib/analyticsClient.ts:1-28`; example usage at `app/src/components/RouteMeta.tsx:1-4`).

Result: the warning is gone, analytics is a separate 5.32 KB raw / 2.11 KB gzip chunk, and the main entry fell from 223.85 KB / 71.55 KB gzip to 218.54 KB / 69.79 KB gzip.

### 4. Wasted home module preload — resolved

The home page preloaded the general site-data chunk even though it is not consumed until deferred below-the-fold content mounts. WebKit warned that the preload was unused. Home now skips that preload while content routes retain it (`app/scripts/prerender-seo.mjs:878-887`). The hero image remains eager, responsive, and high priority (`app/src/components/editorial/QuietHero.tsx:76-105`).

### 5. Accessibility and text-boundary defects — resolved

- The visually duplicated marquee used a second full set of router links. It now renders inert spans for the `aria-hidden` duplicate, removing 16 duplicate interactive elements while preserving the seamless animation (`app/src/components/editorial/QuietHero.tsx:169-188`).
- Explicit whitespace was added after semantic line breaks where browser text extraction and assistive technology otherwise joined adjacent words. The fix covers the brand line, 404, nationwide, services, library, Spanish, tech-audit, and thank-you copy.
- The hero’s visible character cascade already exposes a clean sentence through its accessible H1 label (`app/src/components/editorial/QuietHero.tsx:109-120`).

### 6. Motion/compositor cleanup — resolved

The site already has extensive purposeful motion: headline cascade, marquee, hover/press states, ambient/canvas instruments, count-ups, neon effects, and Safari parallax. Adding more entrance animation would increase mobile work without clarifying the experience. Instead, sections whose reveal animation is intentionally disabled now clear `will-change`, so the browser does not retain unnecessary compositor layers (`app/src/styles/editorial/reveal-static.css:19-46`). The already-prerendered hero likewise clears its character-animation hint (`app/src/components/editorial/QuietHero.css:187-193`). Reduced-motion behavior and existing interaction animation remain intact.

### 7. Dead runtime code and stale truth removed — resolved

The pass removed 418 net lines before this report, including:

- Two unreachable page modules (`Industries.tsx`, `Studio.tsx`) whose routes intentionally redirect to existing sections. Industry and studio detail routes remain.
- An unused command-palette event bridge and subscription. Cmd/Ctrl-K and `/` keyboard activation remain in the real palette.
- An unused daily publishing heatmap model; the monthly publishing visualization that is actually rendered remains.
- Unused `ownerAnswers`, `proofSignals`, `businessTypes`, icon imports, helper exports, and type exports.
- Stale documentation claiming three undeployed functions. At this audit
  checkpoint the repository contained one voice function and its custom health
  route returned HTTP 200. The post-audit release later retired and removed that
  final voice function; see the amendment at the top of this file.

The remaining unused-code scanner findings are build-tool false positives: external public browser scripts, CSS side-effect imports, a declaration shim, types, and data consumed by the prerender script. No confirmed unused active runtime module remains.

### Content and conversion opportunities

These are improvements, not breakages:

1. `/zh/` has the thinnest metadata and both `/zh/` and `/es/` lack a strong above-fold CTA according to the scanner. Add a localized phone/text action and fuller Chinese description after native-language copy review.
2. Five titles are longer than the scanner’s preferred range: `/examples/`, one answer guide, Bronx, Staten Island, and the VenueCircuit case study. Shortening them is optional; current titles are accurate and the pages render correctly.
3. `/glossary/crm/` has a thin description. Expand it with one owner-facing example rather than adding filler.
4. Answer-guide and journal families intentionally reuse a small set of visual archetypes. The next visual upgrade should add two or three subject-specific image families, especially for comparison and emergency articles, rather than adding more global animation.
5. Analytics is delayed but not consent-gated. That is a product/privacy-policy review item, not a runtime defect; legal requirements should be reviewed before changing collection behavior.

## Remaining engineering debt and recommended next work

### Performance architecture

Post-deploy performance is healthy. Netlify's mobile production check scored 91 Performance and 100 for Accessibility, Best Practices, SEO, and PWA. The five-route desktop Lighthouse rail scored 99–100 Performance and 100 for the other categories; home LCP was 785 ms with CLS below 0.001. Unlighthouse scored the five priority routes 84–96 Performance and 100 for Accessibility, Best Practices, and SEO.

The main remaining ceiling is architectural: prerendered SEO markup is painted, then React uses `createRoot` rather than hydrating it (`app/src/main.tsx:28-53`). The next material performance project is true SSG plus `hydrateRoot`, or partial/island hydration. It is not a safe micro-patch because the generated HTML must exactly match the first React render across all 182 routes.

### Deployment-inactive legacy tree

There are 107 tracked HTML files outside `app/`. Netlify publishes only `app/dist`, so these files do not ship (`netlify.toml:12-16`; `SOURCE_OF_TRUTH.md:30-49`). They are repository-confusion debt, not live dead weight. Do not mass-delete them casually: the source-of-truth document records an archive branch and historical recovery path (`SOURCE_OF_TRUTH.md:56-63`). A separate, reviewed archival cleanup is recommended.

### Audit harness dependencies

The production app and the audit harness’s production dependency set both report 0 vulnerabilities. The site-lab harness retains 22 moderate, dev-only advisories in old Lighthouse/Unlighthouse OpenTelemetry and UUID dependency chains. The available automatic resolutions are breaking downgrades/upgrades, so `npm audit fix --force` was intentionally not used. Upgrade the harness as a separately validated tooling task.
